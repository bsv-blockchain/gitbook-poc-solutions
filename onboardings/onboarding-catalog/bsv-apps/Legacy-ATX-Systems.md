## 1. What Is Legacy ATX Systems?

Legacy ATX Systems is a bridge application that enables traditional address-based BSV payments using modern BRC-100 compatible wallets. It provides a simple interface for users who need backward compatibility with legacy Bitcoin address systems while using contemporary wallet infrastructure.

**Key Features:**
- **Address-Based Payments** - Send and receive BSV using traditional Bitcoin addresses
- **BRC-100 Wallet Integration** - Works with modern wallet standards
- **Simple Interface** - Easy send/receive functionality
- **Transaction History** - Track all address-based transactions
- **Backward Compatibility** - Bridge between legacy and modern BSV ecosystems
- **Balance Display** - View your current BSV balance
- **Quick Transfers** - Fast address-to-address payments

**Target Users:** Users needing backward compatibility with legacy systems, those receiving payments from traditional Bitcoin addresses, and anyone bridging between old and new BSV wallet standards.

## 2. Before You Begin

### Prerequisites
- **BSV Wallet:** BRC-100 compatible wallet (Metanet Desktop recommended)
- **Web Browser:** Modern browser with JavaScript enabled
- **BSV Balance:** Some BSV for sending transactions (optional for receiving)
- **Bitcoin Addresses:** Understanding of traditional Bitcoin address format

### Supported Platforms
- **Web-based:** Access via https://legacy.atx.systems/
- **Cross-platform:** Works on desktop and mobile browsers
- **No Installation Required:** Pure web application

### Understanding Legacy Addresses
Legacy ATX bridges two systems:
- **Legacy Addresses:** Traditional format like `1A1zP1eP5QGefi2DMPTfTL5SLmv7DivfNa`
- **Modern Wallets:** BRC-100 standard with enhanced features
- **Compatibility:** Enables transactions between both systems

## 3. Getting Started: Step-by-Step

### Step 1: Access Legacy ATX Systems
1. Navigate to **https://legacy.atx.systems/** in your web browser
2. Ensure your BSV Desktop wallet is running in the background
3. The dashboard displays your address and balance information

![Legacy ATX dashboard showing balance and address](../../../assets/onboardings/bsv-apps/legacy-atx-app/dashboard.png)

### Step 2: Connect Your Wallet
1. Legacy ATX automatically detects your BRC-100 compatible wallet
2. Grant permission for Legacy ATX to access your wallet
3. Your legacy-compatible address will be displayed
4. Current BSV balance appears in the dashboard

### Step 3: Receive BSV to Your Legacy Address
1. **Copy your address** from the dashboard
2. Share this address with the sender
3. Sender can use any Bitcoin wallet to send BSV to your address
4. Wait for blockchain confirmation (typically seconds)
5. Your balance updates automatically

**To receive payments:**
- Click the copy icon or select your address
- Share address via email, message, or QR code
- Monitor dashboard for incoming transactions

### Step 4: Send BSV from Legacy Address
1. Click **"Send"** or navigate to send section
2. Enter the **recipient's Bitcoin address**
3. Input the **amount in BSV** to send
4. Review the transaction details and fee
5. Click **"Send"** to initiate the transaction
6. Confirm the payment in your BSV wallet
7. Transaction is broadcast to the blockchain

![Send and receive workflow interface](../../../assets/onboardings/bsv-apps/legacy-atx-app/send-receive.png)

### Step 5: View Transaction History
1. Access the **"History"** or **"Transactions"** section
2. View all incoming and outgoing transactions
3. Each entry shows:
   - Transaction ID
   - Amount sent or received
   - Timestamp
   - Confirmation status
4. Click on transactions for more details

## 5. Troubleshooting & FAQs

### Common Issues

| Problem | Cause | Solution |
|---------|-------|----------|
| No transaction history showing | Wallet not connected or no transactions | Ensure wallet is connected; verify address has transaction history |
| Cannot send BSV | Insufficient balance or wallet not connected | Check balance; ensure wallet is running and connected |
| Address not displaying | Wallet connection issue | Refresh page; reconnect wallet; check BSV Desktop is running |
| Transaction not confirming | Network congestion or low fee | Wait for confirmation; use higher fee for urgent transactions |
| Balance incorrect | Blockchain sync issue | Refresh page; wait for blockchain to sync; verify on explorer |

### Important Tips
- **Suitable for backward compatibility:** Use when dealing with legacy Bitcoin systems
- **Verify addresses carefully:** Double-check recipient addresses before sending
- **Keep wallet connected:** Ensure BSV Desktop remains running during operations
- **Monitor confirmations:** Wait for at least one confirmation for important transactions
- **Save transaction IDs:** Keep records of important transactions for reference

## 6. Learn More / Next Steps

### Best Practices
- Use Legacy ATX primarily for backward-compatible transactions
- Prefer modern BSV apps and paymail for new interactions

- Verify all addresses on blockchain explorer when in doubt
- Document important addresses and transaction IDs

### Use Cases
- **Legacy Business Integration:** Accept payments from customers using old Bitcoin wallets
- **Cross-System Transfers:** Move funds between legacy and modern wallets
- **Service Compatibility:** Use with exchanges or services requiring traditional addresses
- **Migration Support:** Transition from legacy to modern BSV systems
- **Backward Compatibility:** Maintain support for older Bitcoin infrastructure
